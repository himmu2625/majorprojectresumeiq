Executive SummaryThe Smart Resume Screening Platform is an AI-powered hiring tool that addresses fundamental limitations in traditional Applicant Tracking Systems (ATS). Conventional ATS tools rely on rigid keyword matching, which frequently eliminates qualified candidates whose resumes use different but semantically equivalent terminology.This platform replaces keyword matching with Sentence-BERT (SBERT) based semantic understanding, enabling the system to grasp the contextual meaning behind resume content. Crucially, the system incorporates an Explainable AI (XAI) layer powered by SHAP, which makes every screening decision transparent and interpretable — both for recruiters and for candidates seeking feedback.Core Value PropositionSemantically matches resumes to job descriptions — not just keywordsExplains every match score with human-readable reasoning via XAIProvides actionable resume feedback for candidatesReduces unconscious bias through structured, objective scoringSaves recruiters 60–80% of manual screening time2. Core Features & Requirements4.1  Semantic Resume Matching EngineThe heart of the platform. Uses Sentence-BERT to generate dense vector embeddings of both resume content and job description, then computes cosine similarity to produce a contextual match score.FeaturePriorityTech ComponentExpected OutputPDF Resume Upload & ParsingP0 — Criticalpdfplumber + spaCyStructured text sectionsJob Description InputP0 — CriticalText input + NLP tokenizerCleaned JD tokensSBERT Embedding GenerationP0 — Criticalsentence-transformers (MiniLM)Dense 384-dim vectorsCosine Similarity ScoringP0 — Criticalscikit-learn / PyTorchScore: 0–100%Section-wise ScoringP1 — HighspaCy section parserSkills / Exp / Edu scoresMulti-resume Batch UploadP2 — MediumAsync FastAPI queueRanked candidate list4.2  Explainable AI (XAI) LayerEvery score must be accompanied by a plain-English explanation. This is the differentiating feature of the platform. SHAP values are computed at the token/phrase level to identify which parts of the resume contributed positively or negatively to the match.XAI Output RequirementsTop 5 resume phrases that positively drove the match score (shown in GREEN)Top 5 keywords present in JD but missing from resume (shown in RED)Section-level explanation: 'Your skills section matched 78% of required competencies'Plain-English summary paragraph: auto-generated explanation of the overall resultImprovement priority list: ranked actions the candidate should takeConfidence indicator: low / medium / high match confidence level4.3  Resume Feedback SystemTargeted at job seekers. Rather than simply showing a score, the system provides structured, actionable feedback on resume quality and content gaps.Feedback CategoryWhat the System ProvidesSkills Gap AnalysisList of skills in JD not present in resume, with priority rankingKeyword SuggestionsDomain-specific keywords to add, derived from JD semantic analysisSection Strength RatingIndividual scores for Skills, Experience, Education, Summary sectionsFormat & Completeness CheckFlags missing sections, overly short descriptions, formatting issuesIndustry Alignment ScoreHow well the resume vocabulary aligns with the target industryImprovement Action PlanOrdered list of top 3 changes that will have the most score impact4.4  Recruiter DashboardA clean, professional interface for HR teams to manage screening workflows across multiple job openings.Candidate ranking table with sortable score columnsFilter by score threshold (e.g., show only candidates above 70%)Side-by-side candidate comparison viewExport shortlist to CSV / PDF reportScreening history with timestamps and stored explanationsJob opening management — create, edit, archive job posts4.5  Candidate PortalUpload resume and select target job descriptionView detailed match score with section breakdownRead XAI explanation of match resultDownload personalized improvement suggestions as PDFTrack improvement across multiple resume versions5. Edge Cases & Error HandlingEdge CaseScenarioHandling StrategyScanned / Image PDFPDF contains no extractable text (scanned document)Show error: 'Please upload a text-based PDF or use OCR first'Empty resumeResume PDF has very little text (< 100 words)Warning shown; score marked as 'Low Confidence'Non-English resumeResume written in Hindi or other languageDetect language; show notice; partial scoring attemptedExtremely short JDJob description under 50 wordsWarning: 'JD too short for reliable matching; add more detail'Duplicate resume uploadSame PDF uploaded twice for same JDDetect duplicate via hash; show existing resultCorrupted PDFPDF file is damaged or password-protectedError message with instructions to re-uploadHighly generic resumeResume has no specific skills or experience detailsScore returned with note: 'Resume lacks specificity'SBERT model timeoutML model takes > 15 seconds to respondTimeout error with retry option shown to userNo skills matchZero overlap between resume and JDScore of 0–10%; XAI shows full missing skills listVery long resumeResume exceeds 10 pages / 5000 wordsText truncated intelligently; user notifiedMixed file formatsUser uploads .doc instead of .pdfError shown: 'Please convert to PDF and re-upload'Concurrent uploads10 users upload simultaneouslyQueue system handles requests; user sees position indicator9. Appendix — Glossary & References12.1  GlossaryTermDefinitionSBERT / Sentence-BERTA modification of the BERT model that produces fixed-size sentence embeddings using siamese and triplet networksCosine SimilarityA metric to measure the angle between two vectors in high-dimensional space; value of 1.0 = identical meaningSHAPSHapley Additive exPlanations — a game theory approach to explain the output of any ML modelXAIExplainable Artificial Intelligence — making AI decisions interpretable and auditable by humansATSApplicant Tracking System — software used by companies to manage job applications and filter resumesEmbeddingA dense numerical vector representation of text that captures semantic meaningspaCyAn open-source NLP library for Python, used here for entity recognition and text segmentationFastAPIA modern Python web framework for building APIs; used as the ML model serving layerpdfplumberA Python library for extracting text and metadata from PDF filesP0 / P1 / P2Priority levels: P0 = must have, P1 = should have, P2 = nice to have12.2  Referenced ResearchReimers, N. & Gurevych, I.
